# v0.1.0-preview — Local mech-interp at MBP scale

**Release date:** 2026-05-27  
**Status:** pre-release / preview (not production 1.0)

---

## Summary

v0.1.0-preview is the first public snapshot of a local, agent-driven mechanistic
interpretability research platform designed to run end-to-end on a single Apple Silicon
MacBook Pro — no GPU cluster required. The platform ships 14 experiment families, a
55-command CLI, a Cockpit web UI, and a closed-loop iterate-from-run pipeline that
proposes follow-up experiments automatically. Two findings are included as publishable
artifacts: SAE feature dictionaries are not seed-stable at the 0.9-cosine threshold
(live-only median cosine 0.50 at layer 0, 0.32 at layer 6), and the standard
Arditi/RepE abliteration recipe is formally rejected on Qwen2.5-1.5B-Instruct
(circuit faithfulness 0.041).

---

## Headline findings

### SAE Replication Crisis

*Full report: [`docs/publications/sae_replication_crisis.md`](../publications/sae_replication_crisis.md)*

We trained 5 Top-K Sparse Autoencoders with identical hyperparameters and different
random seeds on GPT-2 small, then used optimal bipartite matching (Hungarian algorithm)
to measure how well the learned feature dictionaries agree across seeds. Restricting to
live (non-dead) features, the median best-match cosine similarity is **0.50 at layer 0**
and **0.32 at layer 6**. With a 512-feature dictionary at layer 6 the median drops
further to **0.26**. The community threshold for "same feature" is cosine ≥ 0.9; **no
condition crosses that threshold** — the stability fraction at ≥ 0.9 is 0.00% in every
layer-6 test and 0.48% at layer 0.

The pattern is counterintuitive: deeper layers and larger dictionaries produce *less*
seed-stability, consistent with a degenerate-basis interpretation. The implication is
that "Feature 47 of the layer-8 SAE detects X" is a property of a specific training
run, not of the underlying model. Auto-interp labels, crosscoder comparisons, and SAE
evaluation benchmarks that treat features as durable handles all inherit this noise.
This result is preliminary (1 model, ~1k training tokens); the platform's reproduce.sh
allows scaling to 1200 runs overnight on A100.

### Qwen Abliteration Failure

*Full report: [`docs/investigations/refusal_audit.md`](../investigations/refusal_audit.md)*

We ran a 4-stage mechanistic audit of refusal in Qwen2.5-1.5B-Instruct (28 layers,
d_model=1536, Apache 2.0). Stage 1 extracted a refusal direction at layer 10 with
projection margin **4.105** — well above the linear-separability threshold of 1.0.
Stage 2 (multi-layer CAA sweep across layers 6/8/10/12) found that steering
effectiveness is decoupled from extraction quality: layer 12 has the highest quality
(4.25) but zero behavioral shift under steering. Stage 3 (circuit patching, 18 forward
passes) found the refusal signal clearly in residual stream at layers 10–11
(recovery 0.50–1.04) but *not* in the local attention head outputs (max recovery 0.13).
Stage 4 (causal scrubbing) formally tested the hypothesis that layers 9–10 attention
heads implement refusal: **faithfulness = 0.041** (chance level), hypothesis **rejected**.

The standard abliteration recipe (Arditi et al., 2024) assumes the refusal direction is
written by attention heads near its most-separable layer. This audit shows that
assumption is unsupported for this model. The refusal information is distributed — most
likely through MLP contributions upstream of layer 10 — and is mechanistically harder
to remove than the recipe assumes. This is a genuine negative result, and the platform
exposes all four run artifacts (IDs 70–73) with full environment.json provenance.

## Capabilities

**Experiment families (14)**

- `circuit_patching` — activation patching with noising baselines
- `acdc_lite` — node-level ACDC automatic circuit discovery on GPT-2 small
- `acdc_edge` — edge-level ACDC with KL-weighted source ablation
- `polysemanticity_sae` — Top-K SAE training + feature analysis
- `attribution_patching` — gradient × activation attribution across layers
- `causal_scrubbing` — hypothesis-driven causal scrubbing pipeline
- `refusal_direction` — Arditi / RepE refusal direction extraction (CAA)
- `caa_steering` — multi-layer contrastive activation addition steering
- `logit_lens` / `tuned_lens` — residual stream layer-by-layer prediction
- `sparse_probing` — linear probing on residual stream activations
- `sae_cross_model` — cross-model SAE feature alignment
- `direct_logit_attribution` — DLA per-head and per-MLP decomposition
- `crosscoder` — cross-model SAE feature comparison (crosscoder architecture)
- `cross_model_representation_probe` — probing shared representations across models

**CLI (55 commands)**

| Group | Commands |
|---|---|
| Core run | `run`, `runs`, `archive-runs`, `summarize-runs`, `report-runs`, `preflight`, `inspect-run`, `export-run` |

| Infra | `config`, `validate`, `providers`, `init-store`, `list-hf-architectures`, `cockpit`, `help`, `demo`, `audit-refusal` |

**Infrastructure**

- 2 backends: TransformerLens (GPT-2/Pythia/Gemma) + universal HuggingFace causal-LM adapter
- Closed-loop `iterate-from-run` with LLM-driven `ProposalGenerator` registry
- Pretrained SAE registry + sae_lens ≥ 3.0 compatibility shim
- Steering vector library (safetensors) with `apply-steering` + composition
- HuggingFace Hub publishing pipeline (`build`, `dry-run`, `upload`)
- Cockpit web UI (FastAPI + HTMX) with 13 routes across home/queue/runs/features/circuit/lens/CAA/refusal/reports/timeline/compare/artifacts/browser views
- Gradio interactive demo (`mech cockpit` / `gradio_app.py`)
- `mech demo` 19-second quickstart (no model download required)
- mkdocs-material GitHub Pages documentation site with `--strict` build gate
- 527 test functions across 58 test files; ruff + mypy --strict clean across 79 source files
- Per-run `environment.json` reproducibility receipts (torch/numpy/transformers versions, uv.lock SHA-256, seed, model weight hash)
- Security: `trust_remote_code` requires explicit opt-in; dunder-path traversal blocked in artifact paths

## Known gaps

- **Multi-model abliteration audit:** Infrastructure for auditing 8 model candidates
  is shipped (`scripts/run_multi_model_audit.py`, `mech audit-refusal`), but only
  Qwen2.5-1.5B-Instruct has been audited end-to-end. The other 7 candidates
  (gpt2-{medium,xl}, pythia-{160m,1.4b}, Llama-3.1-8B, Gemma-2-2B, Mistral-7B)
  are queued; results will land in 0.1.0.
- **IOI circuit reproduction:** The indirect-object identification scaffold and
  `grade_ioi.py` grader are shipped; ACDC runs replicating Wang et al. (2022) are
  still in flight. Canonical numbers will land in 0.1.0.
- **GPU/CUDA support not validated:** All experiments in this release ran on CPU
  (and a limited MPS smoke-test). CUDA path exists in the codebase but has not
  been tested end-to-end. Users on CUDA machines should treat GPU acceleration
  as experimental.
- **SAE seed-stability result is small-scale:** 1 model, 5 seeds, ~1k training
  tokens. Scaling to 10⁶ tokens and 3+ models is needed before claiming generality.
- **Causal scrubbing on Qwen is incomplete:** Stage 3 did not patch MLP outputs
  (only residual-stream and attention-output hook sites). A full per-head + MLP
  follow-up pass is needed to localize the actual refusal writer.

## Reproducibility

**SAE Replication Crisis**

```bash
# From a fresh clone:
bash docs/publications/sae_replication_artifacts/reproduce.sh
# Then open notebooks/06_sae_replication_crisis.ipynb
```

Wall-clock: ~5 minutes on 2024-era Apple Silicon MBP.

**Qwen Abliteration Failure**

```bash
uv sync --group dev --extra interp
uv run mech init-store
uv run mech run --name refusal-direction-qwen     # Stage 1: ~62 min
uv run mech run --name caa-steering-qwen          # Stage 2: ~4 hr
uv run mech run --name refusal-circuit-qwen       # Stage 3: ~3 min
uv run mech run --name causal-scrubbing-refusal-qwen  # Stage 4: ~2 min
uv run mech audit-refusal --refusal-run 70 --caa-run 71 --circuit-run 72 --scrub-run 73
```

Wall-clock: ~4.5 hours CPU on Apple Silicon MBP (Stage 2 dominates).

All runs produce `environment.json` artifacts; run `mech inspect-run <id>` to verify
provenance before citing numbers.
